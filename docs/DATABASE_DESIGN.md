# Database Abstraction Layer Design

## Overview

Berengario uses a database abstraction layer that supports both **SQLite** (for development/simple deployments) and **MariaDB/MySQL** (for production/containerized deployments).

This design ensures:
- Easy local development with SQLite (no setup required)
- Production-ready with MariaDB for containers and scalability
- Seamless switching via configuration
- Database migrations support with Alembic

## Database Selection

### SQLite (Default)
**Use Cases:**
- Local development
- Single-instance deployments
- Simple setups without Docker
- Testing and demos

**Advantages:**
- No setup required
- File-based, portable
- Perfect for development
- Built into Python

**Limitations:**
- No concurrent writes
- Single server only
- Not ideal for high load

**Configuration:**
```bash
DB_TYPE=sqlite
SQLITE_DB_PATH=./data/message_tracker.db
```

### MariaDB/MySQL (Production)
**Use Cases:**
- Docker/Kubernetes deployments
- Multiple instances
- Production environments
- High availability setups

**Advantages:**
- True concurrent access
- Better for containers
- Scalable
- Industry standard

**Configuration:**
```bash
DB_TYPE=mariadb
DB_HOST=mariadb
DB_PORT=3306
DB_NAME=berengario
DB_USER=berengario
DB_PASSWORD=your_secure_password
DB_POOL_SIZE=5
DB_POOL_RECYCLE=3600
```

## Schema Design

### Table: `processed_messages`

Tracks all processed emails to prevent duplicates.

**SQLAlchemy Model:**
```python
class ProcessedMessage(Base):
    __tablename__ = "processed_messages"

    message_id = Column(String(255), primary_key=True)
    sender = Column(String(255), nullable=False, index=True)
    subject = Column(String(500))
    processed_at = Column(DateTime, default=datetime.utcnow, nullable=False)
    status = Column(String(50), default="success", nullable=False)
    error_message = Column(Text, nullable=True)
    attachment_count = Column(Integer, default=0)
    chunks_created = Column(Integer, default=0)

    __table_args__ = (
        Index('idx_sender_date', 'sender', 'processed_at'),
        Index('idx_processed_at', 'processed_at'),
    )
```

**SQL Schema (auto-generated by SQLAlchemy):**
```sql
CREATE TABLE processed_messages (
    message_id VARCHAR(255) PRIMARY KEY,
    sender VARCHAR(255) NOT NULL,
    subject VARCHAR(500),
    processed_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    status VARCHAR(50) NOT NULL DEFAULT 'success',
    error_message TEXT,
    attachment_count INTEGER DEFAULT 0,
    chunks_created INTEGER DEFAULT 0,

    INDEX idx_sender_date (sender, processed_at),
    INDEX idx_processed_at (processed_at)
);
```

### 2. `src/email/db_manager.py`
Database connection and session management.

```python
from contextlib import contextmanager
from sqlalchemy import create_engine
from sqlalchemy.orm import sessionmaker, Session
from src.config import settings
from src.email.db_models import Base

class DatabaseManager:
    """Manages database connections and sessions."""

    def __init__(self):
        self.engine = self._create_engine()
        self.SessionLocal = sessionmaker(bind=self.engine)

    def _create_engine(self):
        """Create SQLAlchemy engine based on config."""
        url = settings.get_database_url()

        if settings.db_type == "sqlite":
            return create_engine(
                url,
                connect_args={"check_same_thread": False},
                echo=settings.debug,
            )
        else:
            # MariaDB/MySQL
            return create_engine(
                url,
                pool_size=settings.db_pool_size,
                pool_recycle=settings.db_pool_recycle,
                pool_pre_ping=True,  # Verify connections
                echo=settings.debug,
            )

    def init_db(self):
        """Create all tables."""
        Base.metadata.create_all(bind=self.engine)

    @contextmanager
    def get_session(self) -> Session:
        """
        Context manager for database sessions.

        Usage:
            with db_manager.get_session() as session:
                message = ProcessedMessage(...)
                session.add(message)
                session.commit()
        """
        session = self.SessionLocal()
        try:
            yield session
            session.commit()
        except Exception:
            session.rollback()
            raise
        finally:
            session.close()

# Global instance
db_manager = DatabaseManager()
```

### 3. `src/email/message_tracker.py`
High-level interface for message tracking.

```python
from datetime import datetime, timedelta
from typing import Optional, List, Dict
from sqlalchemy import func
from src.email.db_manager import db_manager
from src.email.db_models import ProcessedMessage, ProcessingStats

class MessageTracker:
    """Track processed messages and provide statistics."""

    def is_processed(self, message_id: str) -> bool:
        """Check if message has been processed."""
        with db_manager.get_session() as session:
            return session.query(ProcessedMessage).filter_by(
                message_id=message_id
            ).first() is not None

    def mark_processed(
        self,
        message_id: str,
        sender: str,
        subject: str,
        attachment_count: int = 0,
        chunks_created: int = 0,
        status: str = "success",
        error_message: Optional[str] = None,
    ) -> None:
        """Mark message as processed."""
        with db_manager.get_session() as session:
            message = ProcessedMessage(
                message_id=message_id,
                sender=sender,
                subject=subject,
                attachment_count=attachment_count,
                chunks_created=chunks_created,
                status=status,
                error_message=error_message,
            )
            session.add(message)
            # Update stats as well
            self._update_stats(session, status == "success")

    def get_stats(self, days: int = 30) -> Dict:
        """Get processing statistics for last N days."""
        # ... implementation

    def cleanup_old_records(self, days: int = 90) -> int:
        """Delete records older than N days."""
        cutoff_date = datetime.utcnow() - timedelta(days=days)
        with db_manager.get_session() as session:
            deleted = session.query(ProcessedMessage).filter(
                ProcessedMessage.processed_at < cutoff_date
            ).delete()
            return deleted
```

## Database Migrations

Using **Alembic** for schema migrations.

### Setup
```bash
# Initialize Alembic (one-time setup)
alembic init alembic

# Create migration
alembic revision --autogenerate -m "Create processed_messages table"

# Apply migrations
alembic upgrade head

# Rollback
alembic downgrade -1
```

### Migration Configuration (`alembic.ini`)
```ini
[alembic]
script_location = alembic
sqlalchemy.url = # Set dynamically from config

[alembic:exclude]
tables = alembic_version
```

### Migration Script (`alembic/env.py`)
```python
from src.config import settings
from src.email.db_models import Base

config.set_main_option('sqlalchemy.url', settings.get_database_url())
target_metadata = Base.metadata
```

## Performance Considerations

### SQLite
- Use WAL mode for better concurrency
- Regular VACUUM for optimization
- Batch inserts for efficiency

### MariaDB
- Connection pooling (default: 5 connections)
- Connection recycling every hour
- Indexes on frequently queried columns
- Regular monitoring of connection pool

## Security

1. **Credentials**: Never log database passwords
2. **SQL Injection**: Prevented by SQLAlchemy ORM
3. **Connection Strings**: Stored in .env only
4. **Database Backups**: Recommended for production
5. **SSL/TLS**: Optional for MariaDB connections

## Monitoring

Track these metrics:
- Database connection pool usage
- Query performance
- Failed connections
- Table sizes
- Daily processing stats

## Troubleshooting

### SQLite Issues
- **Database locked**: Increase timeout, use WAL mode
- **File permissions**: Ensure write access to db file

### MariaDB Issues
- **Connection refused**: Check host/port, container networking
- **Too many connections**: Increase pool_size
- **Slow queries**: Add indexes, check EXPLAIN
